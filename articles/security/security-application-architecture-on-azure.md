# <a name="application-architecture-on-azure"></a>Architecture de l’application dans Azure

Pour aider à sécuriser vos solutions de nuage de Microsoft Azure, une solide base architecturale est critique. Les architectes, les concepteurs et les implémenteurs tous les bénéficient d’une connaissance solide de l’architecture de services et d’applications. Ces connaissances de base vous permet de comprendre tous les composants de vos solutions en nuage et vous permet d’intégrer la sécurité dans tous les aspects de votre conception et d’implémentation.

Nous avons les éléments suivants pour vous aider dans vos investigations architecturales, dessins et modèles :

- Architecture infographics
- Plans d’architecture
- Jeu de symboles de nuage et d’entreprise
- Modèle de plan 3D Visio

## <a name="architectural-infographics"></a>Architecture infographics

Microsoft publie architecturales plusieurs connexes affiches infographics. Ils sont les suivants :

- [Développement d’Applications réelles Cloud](https://azure.microsoft.com/documentation/infographics/building-real-world-cloud-apps/)
- [Mise à l’échelle avec les Services Cloud](https://azure.microsoft.com/documentation/infographics/cloud-services/)

## <a name="architectural-blueprints"></a>Plans d’architecture

Microsoft publie un ensemble de haut niveau [architecturaux](http://aka.ms/azblueprints) montrant comment créer des types spécifiques de systèmes à l’aide des produits Microsoft.
Chaque plan des éléments suivants :

- Fichier plat 2D basé sur Visio 2003 que vous pouvez télécharger et modifier
- Fichier PDF de couleur perspective 3D pour présenter le plan des audiences moins techniques
- Vidéo qui vous guide à travers la version 3D

## <a name="cloud-and-enterprise-symbol-set"></a>Jeu de symboles de nuage et d’entreprise

[Afficher les symboles vidéo de formation et Visio](http://aka.ms/CnESymbolsVideo) , puis [Télécharger le nuage et la valeur de symbole de l’entreprise](http://aka.ms/CnESymbols) afin de créer des documents techniques qui décrivent Azure, Windows Server, SQL Server et bien plus encore. Vous pouvez utiliser les symboles dans les diagrammes d’architecture, supports de formation, des présentations, les feuilles de données, infographics, livres blancs et même des livres de tiers si le livre apprend à utiliser les produits Microsoft. Toutefois, ils ne sont pas conçus pour une utilisation dans les interfaces utilisateur.

## <a name="3d-blueprint-visio-template"></a>Modèle de plan 3D Visio

Les versions 3D des [Projets d’Architecture de Microsoft](http://aka.ms/azblueprints) ont été créées initialement dans un outil non Microsoft. Un nouveau modèle Visio 2013 (et versions ultérieur) expédiées le 5 août 2015 dans le cadre d’un [cours de certification Microsoft Architecture distribuée sur EDX.ORG](../architecture-overview.md#microsoft-architecture-certification-course).

Le modèle est également disponible en dehors du cours.

- [Afficher la vidéo de formation](http://aka.ms/3dBlueprintTemplateVideo) premier afin de savoir ce qu’il peut faire
- Téléchargez le [Microsoft 3d modèle Visio de plan](http://aka.ms/3DBlueprintTemplate)
- Télécharger [nuage et les symboles de l’entreprise](../architecture-overview.md#drawing-symbol-and-icon-sets) à utiliser avec le modèle 3D
