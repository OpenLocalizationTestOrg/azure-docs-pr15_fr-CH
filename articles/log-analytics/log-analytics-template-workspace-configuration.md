# <a name="manage-log-analytics-using-azure-resource-manager-templates"></a>Gérer l’Analytique de journal à l’aide des modèles du Gestionnaire de ressources Azure

Vous pouvez utiliser [modèles du Gestionnaire de ressources Azure] (... / azure-resource-manager/resource-group-authoring-templates.md) pour créer et configurer des espaces de travail Analytique du journal. Voici quelques exemples des tâches que vous pouvez effectuer avec les modèles :

+ Créer un espace de travail
+ Ajout d’une solution
+ Créer des recherches enregistrées
+ Créer un groupe d’ordinateurs
+ Activer la collecte des journaux IIS à partir d’ordinateurs avec l’agent Windows
+ Collecter les compteurs de performance à partir d’ordinateurs Linux et Windows
+ Collecter les événements de journal système sur les ordinateurs Linux 
+ Collecter des événements à partir des journaux des événements Windows
+ Collecter les journaux des événements personnalisés
+ Ajouter l’agent analytique du journal à une machine virtuelle Azure
+ Configuration analytique de journal pour indexer les données collectées à l’aide des diagnostics de Windows Azure


Cet article fournit des exemples qui illustrent la configuration que vous pouvez effectuer à partir de modèles un modèle.

## <a name="create-and-configure-a-log-analytics-workspace"></a>Créer et configurer un espace de travail journal Analytique

L’exemple de modèle suivant illustre comment :

1.  Créer un espace de travail
2.  Ajouter des solutions à l’espace de travail
3.  Créer des recherches enregistrées
4.  Créer un groupe d’ordinateurs
5.  Activer la collecte des journaux IIS à partir d’ordinateurs avec l’agent Windows
6.  Collecter les compteurs de performance disque logique à partir d’ordinateurs de Linux (% Inodes utilisés ; Méga-octets libres ; Pourcentage utilisé de l’espace ; Transferts disque/s ; Lectures disque/s ; Écritures disque/s)
7.  Collecter des événements de journal système à partir d’ordinateurs de Linux
8.  Collecter les événements d’erreur et d’avertissement dans le journal des événements à partir d’ordinateurs Windows
9.  Collecter le compteur de performance mégaoctets disponibles de mémoire à partir d’ordinateurs Windows
10. Collecter un journal personnalisé 
11. Collecter les journaux IIS et les journaux d’événements Windows écrits par les diagnostics de Windows Azure dans un compte de stockage

### <a name="deploying-the-sample-template"></a>Déploiement de l’exemple de modèle

Pour déployer l’exemple de modèle :

1. Enregistrer l’exemple joint dans un fichier, par exemple`azuredeploy.json` 
2. Modifier le modèle pour que la configuration souhaitée
3. Permet de déployer le modèle de PowerShell ou la ligne de commande

#### <a name="powershell"></a>PowerShell

`New-AzureRmResourceGroupDeployment -Name <deployment-name> -ResourceGroupName <resource-group-name> -TemplateFile azuredeploy.json`

#### <a name="command-line"></a>Ligne de commande

```
azure config mode arm
azure group deployment create <my-resource-group> <my-deployment-name> --TemplateFile azuredeploy.json
```
